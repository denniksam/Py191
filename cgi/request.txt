CGI: разбор запроса

API - интерфейс взаимодействия частей программного комплекса
Web-API - то же, только с использованием НТТР протоколов
Главная задача API - обеспечить информационный обмен. 
Какие возможности по передачи данных предоставляет НТТР:

- метод запроса: строка, с которой начинается запрос
   метод доступен через окружение REQUEST_METHOD, позволяет
   использовать как стандартные, так и методы пользователя

- URL-параметры (Query String) доступны в переменной QUERY_STRING
   в "сыром" виде (не разобранные по именам/значениям)
   URL-кодирование заменяет отдельные символы в QUERY_STRING на
   "экранированные" (escaped, quoted) значения, например
   ' ' - %20 ( в старых стандартах '+' ) 
   ' - %27
   ї - %D1%97
   Инструменты декодирования нужно искать в библиотеках
   Вторая особенность - использование имен без значений (url?json)

- заголовки: записи в виде "имя: значение; атрибуты"
   в CGI заголовки превращаются в переменные с префиксом "НТТР_"
   после которого следует имя заголовка в верхнем регистре
   Значение переменной сохраняет исходное форматирование, атрибуты
   остаются частью значения
   НО! некоторые заголовки используют собственные имена переменных:
   CONTENT_LENGTH, CONTENT_TYPE

- тело запроса: произвольные данные, отделенные пустой строкой от 
   заголовков и до конца пакета. Содержимое тела произвольно,
   наличие тела обязывает добавлять заголовки Content-Length
   и Content-Type. В GET-запросах не должно быть тела.
   В CGI-скрипт тело передается через sys.stdin
